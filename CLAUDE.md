# realestate-app

不動産物件管理・検索アプリケーション。

## プロジェクト概要

このプロジェクトは不動産物件の管理・検索・表示を行うウェブアプリケーションです。

## Git 運用ルール

**コードを変更するたびに、必ず GitHub にプッシュすること。**

具体的な手順:

1. 変更をステージングする
   ```
   git add <変更したファイル>
   ```
2. 意味のあるコミットメッセージでコミットする
   ```
   git commit -m "変更内容の簡潔な説明"
   ```
3. GitHub にプッシュする
   ```
   git push origin <ブランチ名>
   ```

### コミットメッセージの規則

- `feat:` 新機能の追加
- `fix:` バグ修正
- `refactor:` リファクタリング（動作変更なし）
- `style:` スタイル・フォーマット変更
- `docs:` ドキュメント変更
- `test:` テスト追加・修正
- `chore:` ビルド設定・依存関係の更新

例: `feat: 物件検索フィルター機能を追加`

### ブランチ戦略

- `main` — 本番環境相当。直接プッシュ禁止。
- `develop` — 開発のベースブランチ。
- `feature/<機能名>` — 新機能開発用。
- `fix/<バグ名>` — バグ修正用。

## 開発ガイドライン

- セキュリティ脆弱性（SQLインジェクション、XSS など）を絶対に導入しないこと。
- コメントは「なぜそうするか（WHY）」が自明でない場合のみ記述する。
- 不要な抽象化・過剰な機能追加は行わない。タスクに必要な最小限の変更のみ行う。
